Programming Case Study - Book Database with ISBN

SKILLSETS: Python

SCOPE: Individual project in which a Library Management System is to be created in Python code and accessed by the user by a command-line interface, the user should be able to do four basic commands ( Read, Create, Delete and Update ).

Click here to view the Python code and here to view the report.


Role and Work Process

The role given was to develop the library management prototype by myself, making the prototype as functional and user-friendly as possible. Creation of any prototype including this starts with putting yourself into one's shoes, in this case, the user of the library management system. By empathising with the user, ideas for extra features can be easily developed like the ones below.


Outcome Highlights

After the user searches a keyword in the database, the program would not only return all books with the keyword but also highlights the keyword in the data using a "colorama" library import.



Outcome Highlights, cont.

Using a chain of classes and variables, a directory header tells the user where are they currently in the program. For example, if they enter the "Book Search" and then enter "Book Search via Book Title", the directory header will show "Library Management Menu > Book Search > Book Search via Book Title" to allow users to know where they are.

Such implementation allows new users to easily learn and navigate through the program.
